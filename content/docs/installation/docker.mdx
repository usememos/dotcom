Docker is the recommended installation method for Memos.

## Quick Start

```bash
docker run -d \
  --name memos \
  --publish 5230:5230 \
  --volume ~/.memos/:/var/opt/memos \
  neosmemo/memos:stable
```

Visit [http://localhost:5230](http://localhost:5230) to access Memos.

## Docker Compose

For production, use Docker Compose:

```yaml title="docker-compose.yml"
services:
  memos:
    image: neosmemo/memos:stable
    container_name: memos
    ports:
      - "5230:5230"
    volumes:
      - ./memos:/var/opt/memos
    restart: unless-stopped
```

```bash
docker compose up -d
```

## With PostgreSQL

```yaml title="docker-compose.yml"
services:
  postgres:
    image: postgres:15
    environment:
      POSTGRES_DB: memos
      POSTGRES_USER: memos
      POSTGRES_PASSWORD: your-password
    volumes:
      - postgres_data:/var/lib/postgresql/data

  memos:
    image: neosmemo/memos:stable
    depends_on:
      - postgres
    environment:
      - MEMOS_DRIVER=postgres
      - MEMOS_DSN=postgresql://memos:your-password@postgres:5432/memos
    ports:
      - "5230:5230"

volumes:
  postgres_data:
```

## Update

```bash
docker compose pull
docker compose up -d
```

<Callout type="info">
Use specific version tags (e.g., `neosmemo/memos:0.24.0`) in production instead of `stable`.
</Callout>
