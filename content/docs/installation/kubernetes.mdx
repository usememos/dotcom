Kubernetes provides robust orchestration for Memos with auto-scaling and high availability.

## Prerequisites

- Kubernetes cluster (1.19+)
- `kubectl` configured
- Storage class for persistent volumes

## Basic Deployment

```yaml title="memos.yaml"
apiVersion: v1
kind: Namespace
metadata:
  name: memos
---
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: memos-data
  namespace: memos
spec:
  accessModes: [ReadWriteOnce]
  resources:
    requests:
      storage: 10Gi
---
apiVersion: apps/v1
kind: Deployment
metadata:
  name: memos
  namespace: memos
spec:
  replicas: 1
  selector:
    matchLabels:
      app: memos
  template:
    metadata:
      labels:
        app: memos
    spec:
      containers:
      - name: memos
        image: neosmemo/memos:stable
        ports:
        - containerPort: 5230
        env:
        - name: MEMOS_MODE
          value: "prod"
        volumeMounts:
        - name: data
          mountPath: /var/opt/memos
      volumes:
      - name: data
        persistentVolumeClaim:
          claimName: memos-data
---
apiVersion: v1
kind: Service
metadata:
  name: memos
  namespace: memos
spec:
  selector:
    app: memos
  ports:
  - port: 5230
    targetPort: 5230
```

```bash
kubectl apply -f memos.yaml
```

## Helm Chart

A community-maintained Helm chart is available:

```bash
git clone https://github.com/usememos/helm.git
cd helm
helm install memos . --namespace memos --create-namespace
```

## Ingress

```yaml title="ingress.yaml"
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: memos
  namespace: memos
spec:
  ingressClassName: nginx
  rules:
  - host: memos.example.com
    http:
      paths:
      - path: /
        pathType: Prefix
        backend:
          service:
            name: memos
            port:
              number: 5230
```

<Callout type="info">
Use specific version tags (e.g., `neosmemo/memos:0.24.0`) in production instead of `stable`.
</Callout>
