Memos supports rich content formatting through **CommonMark** and **GitHub Flavored Markdown (GFM)** with a custom tag syntax. This guide covers all supported formatting options.

## Basic Text Formatting

### Emphasis and Styling

```markdown
**Bold text** or __Bold text__
*Italic text* or _Italic text_
***Bold and italic*** or ___Bold and italic___
~~Strikethrough text~~
`Inline code`

Combine formatting: **bold with *nested italic* text**
```

**Result:**
**Bold text**, *Italic text*, ***Bold and italic***, ~~Strikethrough text~~, `Inline code`

<Callout type="info">
Strikethrough (`~~text~~`) is part of GitHub Flavored Markdown (GFM).
</Callout>

## Headers

Headers create structure and improve readability:

```markdown
# Header 1
## Header 2
### Header 3
#### Header 4
##### Header 5
###### Header 6
```

<Callout type="tip">
Use headers to organize your content hierarchically. Headers automatically generate anchor links for navigation.
</Callout>

## Lists

### Unordered Lists

Create bullet lists using `-`, `*`, or `+`:

```markdown
- First item
- Second item
  - Nested item
  - Another nested item
    - Deeply nested
- Third item

* Alternative syntax with asterisks
+ Alternative syntax with plus signs
```

**Result:**
- First item
- Second item
  - Nested item
  - Another nested item
- Third item

### Ordered Lists

```markdown
1. First item
2. Second item
   1. Nested ordered item
   2. Another nested item
3. Third item

1. Numbers auto-increment
1. So you can use 1 for all
1. And markdown handles numbering
```

**Result:**
1. First item
2. Second item
3. Third item

### Task Lists (GFM)

Create checkable task lists:

```markdown
- [x] Completed task
- [ ] Incomplete task
- [x] Another completed task
  - [ ] Nested incomplete task
  - [x] Nested completed task
```

**Result:**
- [x] Completed task
- [ ] Incomplete task
- [x] Another completed task

<Callout type="success">
Task lists are great for tracking progress in project notes, meeting agendas, and to-do items.
</Callout>

## Links

### Basic Links

```markdown
[Link text](https://example.com)
[Link with title](https://example.com "Tooltip on hover")

Autolinks: https://usememos.com
Email: <contact@example.com>
```

**Result:**
[Link text](https://example.com), https://usememos.com

### Reference-Style Links

For cleaner markdown when using the same link multiple times:

```markdown
[Memos documentation][docs]
[Another reference to docs][docs]

[docs]: https://usememos.com/docs "Memos Docs"
```

### Internal Links

Link to other sections or pages:

```markdown
[Getting Started Guide](/docs/guides/getting-started)
[Link to specific section](#headers)
```

## Images

### Basic Image Syntax

```markdown
![Alt text](https://example.com/image.png)
![Image with title](https://example.com/image.png "Image tooltip")
```

### Linked Images

Make images clickable:

```markdown
[![Screenshot](image.png)](https://example.com)
```

<Callout type="info">
**Image Storage**: Upload images directly to your Memos instance. Supported formats include PNG, JPG, GIF, and WebP.
</Callout>

## Code

### Inline Code

Use backticks for inline code:

```markdown
Use `const` instead of `var` in JavaScript.
Configuration file: `/etc/memos/config.yaml`
Press `Ctrl+S` to save.
```

**Result:**
Use `const` instead of `var` in JavaScript.

### Code Blocks

Create multi-line code blocks with syntax highlighting:

````markdown
```javascript
function greet(name) {
  return `Hello, ${name}!`;
}

const message = greet('World');
console.log(message);
```
````

**Result:**

```javascript
function greet(name) {
  return `Hello, ${name}!`;
}

const message = greet('World');
console.log(message);
```

## Tables (GFM)

Create tables using pipes and hyphens:

```markdown
| Column 1 | Column 2 | Column 3 |
|----------|----------|----------|
| Row 1    | Data     | More data |
| Row 2    | Data     | More data |
| Row 3    | Data     | More data |
```

**Result:**

| Column 1 | Column 2 | Column 3 |
|----------|----------|----------|
| Row 1    | Data     | More data |
| Row 2    | Data     | More data |

### Column Alignment

Control text alignment with colons:

```markdown
| Left Aligned | Center Aligned | Right Aligned |
|:-------------|:--------------:|--------------:|
| Left         | Center         | Right         |
| Text         | Text           | Text          |
```

**Result:**

| Left Aligned | Center Aligned | Right Aligned |
|:-------------|:--------------:|--------------:|
| Left         | Center         | Right         |
| Text         | Text           | Text          |

## Blockquotes

Create quoted text blocks:

```markdown
> This is a blockquote.
>
> It can span multiple paragraphs.

> **Pro Tip:** You can use other markdown inside blockquotes.
>
> - Including lists
> - And other formatting
> - Like **bold** and *italic*
```

**Result:**

> This is a blockquote.
>
> It can span multiple paragraphs.

### Nested Blockquotes

```markdown
> Level 1 quote
>> Level 2 nested quote
>>> Level 3 nested quote
```

## Horizontal Rules

Create visual dividers:

```markdown
Three or more hyphens:

Three or more asterisks:
***

Three or more underscores:
___
```

All produce the same result:

## Tags (Memos Custom Syntax)

Tags are Memos' primary organizational feature. Use the `#` symbol followed by tag name:

```markdown
#work #project #urgent
#personal #ideas #journal
#tech #programming #golang
#meeting #2025 #q1
```

### Tag Syntax Rules

- Tags start with `#` followed by alphanumeric characters
- No spaces allowed (use hyphens or underscores instead)
- Case-sensitive (though lowercase is recommended)
- Can include numbers: `#2025`, `#project-1`

### Tag Organization Strategies

**Flat structure:**
```markdown
#work #home #shopping #ideas
```

**Category-based:**
```markdown
#category-work #category-personal
#type-task #type-note #type-reference
```

**Project-based:**
```markdown
#project-website #project-api #project-docs
```

**Time-based:**
```markdown
#2025 #2025-q1 #january #week-02
```

<Callout type="success" title="Tag Best Practices">
- Use lowercase for consistency
- Create a tagging convention and stick to it
- Avoid creating too many similar tags
- Review and consolidate tags periodically
- Use multi-word tags with hyphens: `#project-alpha`, `#team-meeting`
</Callout>

## HTML Support

Memos supports safe HTML elements through GFM:

### Collapsible Sections

```html
<details>
<summary>Click to expand</summary>

Hidden content goes here.

You can use **Markdown** inside!

</details>
```

**Result:**

<details>
<summary>Click to expand</summary>

Hidden content goes here.

You can use **Markdown** inside!

</details>

### Keyboard Keys

```html
Press <kbd>Ctrl</kbd> + <kbd>S</kbd> to save
Use <kbd>Cmd</kbd> + <kbd>K</kbd> for search
```

**Result:**
Press <kbd>Ctrl</kbd> + <kbd>S</kbd> to save

### Text Formatting

```html
<mark>Highlighted text</mark>
H<sub>2</sub>O (subscript)
x<sup>2</sup> (superscript)
```

**Result:**
<mark>Highlighted text</mark>, H<sub>2</sub>O, x<sup>2</sup>

<Callout type="warn">
Only safe HTML elements are allowed. Scripts and potentially dangerous tags are sanitized for security.
</Callout>

## Line Breaks

### Hard Line Breaks

End a line with two spaces or a backslash:

```markdown
Line one··
Line two (two trailing spaces above)

Line one\
Line two (backslash above)
```

### Paragraph Breaks

Leave a blank line between paragraphs:

```markdown
First paragraph.

Second paragraph.
```

## Escaping Characters

Escape special markdown characters with backslash:

```markdown
\*Not italic\*
\#Not a tag
\[Not a link\](url)
\`Not code\`

Use backslash before: \ ` * _ { } [ ] ( ) # + - . !
```

**Result:**
\*Not italic\*, \#Not a tag

## Tips and Best Practices

### Writing Effective Memos

1. **Start with a clear first line** - This appears in previews and search results
2. **Use headers for structure** - Break up longer content into scannable sections
3. **Tag consistently** - Develop and maintain a tagging system
4. **Format code properly** - Use code blocks with language specification
5. **Link related memos** - Create connections for better knowledge management

### Formatting Tips

1. **Keep it simple** - Don't over-format; focus on content
2. **Use task lists for actionable items** - Track progress directly in memos
3. **Tables for structured data** - Better than trying to format with spaces
4. **Blockquotes for highlights** - Draw attention to important information
5. **Horizontal rules sparingly** - Use to separate major sections

### Performance and Readability

1. **Optimize images** - Compress images before uploading
2. **Break up large tables** - Consider splitting very wide or long tables
3. **Use meaningful alt text** - Helps with accessibility and context
4. **Proper heading hierarchy** - Don't skip heading levels (h1 → h2 → h3)
5. **Limit line length** - Shorter lines are easier to read
