Memos supports Markdown formatting to help you create rich, well-structured content. This guide covers all the syntax you can use in your memos.

## Text Formatting

Make your text stand out with basic formatting:

```markdown
**Bold text**
*Italic text*
***Bold and italic***
~~Strikethrough~~
`Inline code`
```

**Result:**
**Bold text**, *Italic text*, ***Bold and italic***, ~~Strikethrough~~, `Inline code`

You can also combine formatting: **bold with *nested italic* text**

## Headers

Organize your content with headers:

```markdown
# Heading 1
## Heading 2
### Heading 3
#### Heading 4
##### Heading 5
###### Heading 6
```

<Callout type="tip">
Headers create anchor links automatically, making it easy to link to specific sections.
</Callout>

## Lists

### Unordered Lists

Create bullet points with `-`, `*`, or `+`:

```markdown
- First item
- Second item
  - Nested item
  - Another nested item
- Third item
```

**Result:**
- First item
- Second item
  - Nested item
  - Another nested item
- Third item

### Ordered Lists

Create numbered lists:

```markdown
1. First item
2. Second item
3. Third item
```

**Result:**
1. First item
2. Second item
3. Third item

### Task Lists

Track your to-dos:

```markdown
- [x] Completed task
- [ ] Pending task
- [ ] Another task
  - [x] Completed subtask
```

**Result:**
- [x] Completed task
- [ ] Pending task
- [ ] Another task

## Links

### Basic Links

```markdown
[Link text](https://example.com)
[Link with tooltip](https://example.com "Hover to see this")
```

**Result:**
[Link text](https://example.com)

### Auto-linking

URLs and emails are automatically linked:

```markdown
https://usememos.com
<contact@example.com>
```

### Reference Links

For repeated links, use references:

```markdown
Check out [Memos][1] for more info about [Memos][1].

[1]: https://usememos.com
```

## Images

### Basic Images

```markdown
![Description of image](https://example.com/image.png)
![Image with tooltip](https://example.com/image.png "Hover text")
```

### Clickable Images

Make images into links:

```markdown
[![Alt text](image.png)](https://example.com)
```

<Callout type="info">
Upload images directly to your Memos instance. Supported formats: PNG, JPG, GIF, WebP.
</Callout>

## Code

### Inline Code

Use backticks for inline code:

```markdown
Use the `console.log()` function to debug.
The config file is at `/etc/memos/config.yaml`.
```

**Result:**
Use the `console.log()` function to debug.

### Code Blocks

For multi-line code, use triple backticks with the language name:

````markdown
```javascript
function greet(name) {
  return `Hello, ${name}!`;
}
```
````

**Result:**

```javascript
function greet(name) {
  return `Hello, ${name}!`;
}
```

### Supported Languages

````markdown
```bash
echo "Shell script"
```

```python
def hello():
    print("Hello, World!")
```

```go
package main

func main() {
    fmt.Println("Hello")
}
```

```sql
SELECT * FROM memos WHERE id = 1;
```

```json
{
  "name": "memos",
  "version": "0.23.0"
}
```

```yaml
services:
  memos:
    image: neosmemo/memos:stable
```
````

Plus many more: `typescript`, `java`, `rust`, `cpp`, `html`, `css`, `dockerfile`, etc.

## Tables

Create tables with pipes and dashes:

```markdown
| Name    | Role      | Status |
|---------|-----------|--------|
| Alice   | Developer | Active |
| Bob     | Designer  | Active |
```

**Result:**

| Name    | Role      | Status |
|---------|-----------|--------|
| Alice   | Developer | Active |
| Bob     | Designer  | Active |

### Align Columns

Use colons to control alignment:

```markdown
| Left | Center | Right |
|:-----|:------:|------:|
| Text | Text   | Text  |
```

**Result:**

| Left | Center | Right |
|:-----|:------:|------:|
| Text | Text   | Text  |

## Quotes

Create blockquotes with `>`:

```markdown
> This is a quoted text.
>
> You can have multiple paragraphs.

> **Tip:** Use quotes to highlight important information.
```

**Result:**

> This is a quoted text.
>
> You can have multiple paragraphs.

### Nested Quotes

```markdown
> First level
>> Second level
>>> Third level
```

## Horizontal Lines

Create dividers with three or more dashes, asterisks, or underscores:

```markdown
---
```

**Result:**

---

## Tags

Organize your memos with tags using `#`:

```markdown
#work #project #urgent
#personal #ideas
#tech #programming
```

Tags help you categorize and find your memos easily.

### Tagging Tips

**Simple tags:**
```markdown
#work #home #ideas
```

**Category system:**
```markdown
#work-meeting #work-project
#personal-journal #personal-finance
```

**Date-based:**
```markdown
#2025 #january #week-1
```

<Callout type="success" title="Best Practices">
- Keep tags lowercase for consistency
- Use hyphens for multi-word tags: `#project-alpha`
- Create a tagging system and stick to it
- Review tags regularly to avoid duplicates
</Callout>

## HTML Elements

Some HTML elements are supported for advanced formatting.

### Collapsible Sections

```html
<details>
<summary>Click to expand</summary>

Hidden content here. You can use **Markdown** inside!

</details>
```

**Result:**

<details>
<summary>Click to expand</summary>

Hidden content here. You can use **Markdown** inside!

</details>

### Keyboard Keys

```html
Press <kbd>Ctrl</kbd> + <kbd>S</kbd> to save
```

**Result:**
Press <kbd>Ctrl</kbd> + <kbd>S</kbd> to save

### Special Formatting

```html
<mark>Highlighted text</mark>
H<sub>2</sub>O
x<sup>2</sup>
```

**Result:**
<mark>Highlighted text</mark>, H<sub>2</sub>O, x<sup>2</sup>

<Callout type="warn">
Only safe HTML is allowed. Scripts and dangerous elements are blocked for security.
</Callout>

## Line Breaks

### Single Line Break

End a line with two spaces or a backslash:

```markdown
First line
Second line (two spaces above)

First line\
Second line (backslash above)
```

### Paragraph Break

Leave a blank line:

```markdown
First paragraph.

Second paragraph.
```

## Escaping

Use backslash to display special characters literally:

```markdown
\*Not italic\*
\#Not a tag
\[Not a link\]
```

**Result:**
\*Not italic\*, \#Not a tag, \[Not a link\]

## Tips for Great Memos

**Keep it organized**
- Start with a clear first line (appears in previews)
- Use headers to structure longer memos
- Break up walls of text with lists and spacing

**Use tags wisely**
- Create a consistent tagging system
- Don't over-tag (3-5 tags per memo is usually enough)
- Use specific tags for important projects

**Format for readability**
- Use code blocks for commands and code snippets
- Add task lists for action items
- Use tables for structured data
- Use quotes to highlight key points

**Optimize media**
- Compress large images before uploading
- Use descriptive alt text for images
- Keep file sizes reasonable
