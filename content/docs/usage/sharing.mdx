Sharing in Memos is controlled primarily through memo visibility.

## Visibility levels

- `PRIVATE`: only you can read the memo
- `PROTECTED`: authenticated users on the instance can read it
- `PUBLIC`: anyone with the link or public access to the instance can read it

## Safe sharing habits

- choose visibility intentionally before posting
- avoid making operational or sensitive notes public by default
- review attachments before publishing a memo openly
- disable public memo features entirely if your instance is private

## Explore and discoverability

Public content is easier to discover and share, which is useful for team knowledge bases or public writing. It also raises the bar for review, tagging, and privacy hygiene.

## Typical use cases

- `PRIVATE` for drafts and personal notes
- `PROTECTED` for team-internal knowledge
- `PUBLIC` for public docs, announcements, and openly shared references
